Instruções para o Desenvolvimento em Equipe no Github

Passo 1 - Ao acessar o Git Bash de dentro da pasta do projeto, entre no branch master
> git checkout master

Passo 2 - Atualize a pasta do seu computador com o repositório online (para você ter tudo o que pode ter sido enviado por outro integrante da equipe)
> git pull origin master

Passo 3 - Crie a sua Branch de modificação a partir da versão master do repositorio. O nome dabrnch nao pode conter espacos.
> git checkout -b Nome_da_Branch master

Passo 4 - Feche o Git Bash e faca o seu trabalho, abrindo a Unit pela pasta do projeto (onde você abriu o Git Bash). Ao terminar, salve o projeto na Unit e reabra o Git Bash da pasta do projeto.

Passo 5 - Adicione tudo o que foi feito
> git add . 

Passo 6 - Confirme as suas alteracoes tornando-as permanentes
> git commit -m "Descreva brevemente as alteracoes realizadas"

Passo 7 - Envie para o servidor Github. Se for o primeiro push desta branch, digite o codigo completo abaixo. Caso ja tenha sido feito algum push antes, nessa branch, digite apenas git push.
> git push --set-upstream origin Nome_da_Branch
